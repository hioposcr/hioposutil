# Inicio rápido

Esta página sirve para personas que necesitan empezar sin leer todo el manual desde el principio.

## ¿Qué tipo de caso tiene?

Use esta guía rápida:

| Si le pasa esto | Use este módulo |
| --- | --- |
| Tiene un solo documento rechazado y quiere corregirlo | `Corrección individual` |
| Tiene varios XML para reprocesar en secuencia | `Reenvío masivo` |
| Necesita cambiar terminal y renumerar muchos documentos | `Reenvío masivo` |
| Quiere revisar un solo caso antes de reenviar | `Corrección individual` |

![Ruta rápida de decisión](assets/mapa-general.svg)

## Camino 1: tengo un solo documento

Siga este orden:

1. Cargue el XML original.
2. Cargue la nota rechazada.
3. Analice el documento.
4. Recalcule el ajuste.
5. Genere la versión corregida.
6. Revise el resultado.
7. Envíe a MDG si corresponde.

## Camino 2: tengo varios XML

Siga este orden:

1. Cargue los XML del lote.
2. Defina nueva terminal.
3. Defina número inicial.
4. Revise la configuración MDG.
5. Prepare el lote.
6. Revise la cola.
7. Envíe el lote a MDG.

## Qué debe tener listo antes de comenzar

- los archivos correctos del caso
- la terminal que desea usar
- las credenciales MDG del cliente correcto
- claridad sobre si trabajará en `Testing` o `Producción`

> Consejo: si siente duda sobre cuál módulo usar, empiece por esta página y luego entre al módulo específico.
